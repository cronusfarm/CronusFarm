TriGorilla(Mega2560) + RepRapDiscount Smart Controller 2004A

실크(라벨) 기준 커넥터/점퍼 표기 + Arduino Mega 핀 매핑 요약
대상 TriGorilla V1.x(8-bit ATmega2560) 계열

1) TriGorilla 보드 종류와 특징(버전 주의)

  • V1.0 / V1.1: 가장 흔함, 8비트 ATmega2560 기반. (Marlin 기준 보통 BOARD_TRIGORILLA_14)
  • V0.0.2 / V0.0.3: 전원 단자 색상/서보 배치 일부 변경.
  • V3.0.x: 32비트(HC32F460) 기반. 핀 할당이 완전히 달라서 본 문서의 “Mega 핀 매핑”을 그대로 쓰면 안 됨.
주의 버전이 확실하지 않으면, “보드 실크의 커넥터 라벨”을 기준으로 먼저 확인하세요.

2) 전원 입력 / 전환 점퍼

파란색 스크류 2핀(전원 입력)

  • 5A+: 보드 로직/모터드라이버/HEATER0·HEATER1·FAN 전원(필수)
  • 11A+: +HOTBED- 전용 고전류 입력(히팅베드 쓸 때만)

DC/USB 전환 3핀 점퍼

  • USB: 업로드/점검용(USB 5V 로직 구동)
  • DC: 실사용(외부 전원으로 구동)

3) 고출력/팬 단자(MOSFET Outputs)

파란색 스크류(출력)

단자Arduino Mega 핀설명
+HEATER0-D10가장 강력한 출력(대용량 펌프/히터 등)
+HEATER1-D7보조 고출력
+HOTBED-D811A 입력과 연동(고전류)

흰색 2핀(FAN)

  • +FAN0-: D9 (PWM 가능)
  • +FAN1- / FAN2-: 상시 전원(Always ON)

4) 모터/드라이버

  • 하얀색 4핀 커넥터(표기): X, Y, Z, Z, E0, E1
  • 40P*2 암컷 소켓: X/Y/Z/E0/E1 드라이버 장착(A4988, TMC2208 등)

5) 확장/점퍼(라벨 기준)

  • END STOPS (3핀, S/G/V): Z+, Z-, Y+, Y-, X+, X-
  • D42 / D43 (3핀): 신호 / GND / 5V
  • UART3 (4핀): GND / RX3 / TX3 / 5V
  • ISP (3×2핀): 1~6
  • IIC (6핀): 12V / GND / SCL / SDA / GND / 5V
  • T0~T3 (4×2핀): 온도 센서(써미스터)
  • EXP1/EXP2 (10핀 플랫): 2004A 패널 연결

2. D42, D43 핀의 기능

보드상에 독립적으로 인쇄된 D42, D43은 주로 확장성을 위해 배치된 범용 디지털 I/O(GPIO) 핀입니다. TriGorilla 보드 설계상 이 핀들은 특정 기능이 고정되어 있지 않아 사용자 용도로 활용됩니다.
  • 주요 활용: 필라멘트 감지 센서(Filament Runout), 자동 베드 레벨링(ABL/BL-Touch), 사용자 정의 스위치/LED/릴레이 제어
  • 특징: 5V 로직. Arduino에서 digitalWrite(42, HIGH), digitalRead(43)처럼 번호 그대로 사용 가능
구분핀 번호주요 용도
T0~T3A13, A14, A15, A12온도 측정(100K 서미스터 연결)
D42Digital 42필라멘트 센서 또는 범용 출력
D43Digital 43확장 센서 또는 범용 출력

7) 전체 시리얼 포트 위치 요약(보드 라벨 기준)

TriGorilla(Mega2560 기반)는 하드웨어 시리얼이 4개(Serial0~3)이며, “어디에 물려 있는지”가 업로드/통신 안정성에 직접 영향이 있습니다.
시리얼 핀 번호 보드상 물리적 위치(라벨)
Serial 0 D0(RX), D1(TX) USB 연결 포트(PC/Pi 업로드·통신용, 기본 업로드 포트)
Serial 1 D19(RX1), D18(TX1) END STOPS의 Z- 및 Z+ 포트의 Signal 핀
Serial 2 D17(RX2), D16(TX2) EXP1 커넥터(LCD 통신과 공유됨)
Serial 3 D15(RX3), D14(TX3) UART3 라고 적힌 별도의 4핀 커넥터
권장 USB 업로드(Serial0)와 UART 통신이 충돌하지 않도록, UART는 Serial1 등 업로드 경로와 다른 하드웨어 시리얼을 쓰세요.

CronusFarm 펌웨어 기준(현재): Serial1(D18/D19) 고정

  • Mega TX1 D18 → R4 RX D0 / Mega RX1 D19 ← R4 TX D1 (교차), GND 공통
  • TriGorilla/RAMPS 계열에서는 위 핀이 END STOP의 Z− / Z+ 쪽 Signal(S) 과 같은 배선인 경우가 많습니다. UART3(D14/D15) 4핀 헤더는 FAN Y+/Y−와 같은 핀이라 R4 연결에 쓰지 마세요.
  • SCL/SDA(IIC) 미사용 이유: 2026-04 기준 실환경에서 I2C는 케이블 길이/노이즈/부팅 타이밍 영향으로 패널이 로컬 환영 3줄에 머물고 엔코더 이벤트가 유실되는 사례가 반복되었습니다. UART(Serial1)는 배선·장애 재현성이 높아 원격 업로드 운영과 함께 표준으로 고정합니다.
  • FAN 4채널(마스크 하위 4비트): X−/X+/Y−/Y+ ENDSTOP S → D3,D2,D14,D15. 팬이 3개면 그중 3비트만 사용하면 됩니다.

8) Pi에서 Mega USB 업로드가 “됐다/안 됐다” 할 때(원인·대응)

아래는 실제로 관측된 증상(dmesg, avrdude timeout)과 CronusFarm에서 적용한 대응을 요약한 것입니다.

문서 정책: CronusFarm 패널–R4 링크는 Serial1(D18/D19, Z-/Z+ Signal)로 고정되어 있고, Mega 업로드는 USB(CP210×)→부트로더 경로입니다. 업로드 문제를 UART 배선을 잠시 빼서 우회한다는 식의 운영 대응은 본 프로젝트 문서에 적지 않습니다(동일 장비에서 재현 가능한 절차로 두지 않음).

원인 A — USB 링크 자체가 끊김/불안정(EMI·전원·케이블)

  • 증상: dmesg에 cp210x ttyUSB0: failed set request ... -110, usb ... disabled by hub (EMI?), disconnect 같은 로그가 반복.
  • 이유: CP210x 브리지가 USB 제어 전송에 실패하면 업로드 중 통신이 끊긴다(소프트웨어만으로 100% 보장 어려움).
  • 대응: Pi에 직결·짧은 데이터 케이블·노이즈 분리·필요 시 usbcore.autosuspend=-1(커널 파라미터) 등 전원/배선 측 조치.

원인 B — 업로드 포트/명령 호출 실패(운영 실수)

  • 증상: Windows에서 SSH로 Pi에 명령을 보낼 때 $PORT 같은 변수가 로컬에서 먼저 비워져 -p 인자가 깨지는 경우가 있다.
  • 대응: Pi에서는 가능하면 포트를 하드코딩하거나, SSH 문자열에서 $ 이스케이프를 확실히 한다. Mega는 /dev/serial/by-id/... 고정이 안정적이다.

10) 개발흐름 표시 규격(운영 기준)

부팅화면 (패널 Mega 자체 출력, 4줄) — arduino/CronusFarmPanel/CronusFarmPanel.ino

  • 0행: CronusFarm Panel (테스트 모드면 CronusFarm Panel TEST)
  • 1행: UART ready
  • 2행: 미출력/이전 내용 유지
  • 3행: 미출력/이전 내용 유지

환영메세지 (패널 Mega 자체 출력, 4줄) — arduino/CronusFarmPanel/CronusFarmPanel.ino

  • 0행: Welcome 2 CronusFarm
  • 1행: gDateLine (Pi가 보내는 DT:...)
  • 2행: gTimeLine (Pi가 보내는 TM:...)
  • 3행: 빈 줄

각 Bed 화면 (UNO R4가 패널 LCD로 출력, 4줄) — arduino/CronusFarm/CronusFarm.ino

  • 환영 화면: 0행 Welcome 3, 1행 CronusFarm, 2행 날짜, 3행 시간(AM/PM)
  • Bed 브라우즈: 0행 A Bed/B Bed/C Bed/D Bed, 1행 MODE:<AUTO|MAN |MIX |N/A >
  • 2행: 채널 있으면 STATE:<ON |OFF|MIX> <켜진개수>/<전체개수>, D Bed는 STATE: -- (reserved)
  • 3행: Bed<현재>/<4> Dial Nxt Push

Bed 채널 매핑(현재 코드 기준)

  • A Bed: LED A1, LED A2, PUMP A1, PUMP A2
  • B Bed: LED B1, PUMP B1, PUMP B2
  • C Bed: FAN A1, FAN B1, FAN B2
  • D Bed: 예비(채널 없음)

9) R4–Mega TTL 연결에서 “5V 미연결”을 어떻게 확인하나

  • 1) 실크/문서로 핀 순서 확인: 커넥터에 5V 또는 VCC 표기가 있는 핀을 먼저 확인한다.
  • 2) 멀티미터로 전압 확인(가장 확실): 보드가 USB 또는 외부전원으로 정상 구동 중일 때, 의심 핀과 인접 GND 사이 전압이 약 +5V면 그 핀은 전원 라인이다. (UART 신호선은 이렇게 뜨지 않는다.)
  • 3) 배선 시각 확인: GND + TX/RX 신호만 연결하고, 어떤 4핀이든 5V 전원 슬롯은 비워 둔다(CronusFarm Mega는 R4를 보통 Z± 스톱 Signal 경로로 물며, 예전처럼 UART3 4핀만이 아닐 수 있음).
  • 왜 중요?: UART는 보통 3.3V/5V TTL 레벨만 신호로 주고받으면 되고, Mega와 R4 사이에 불필요한 5V 상호 공급은 오히려 리스크가 된다.
교차 연결이 맞더라도, 헤더의 어느 핀이 5V인지만 헷갈리면 위 확인 절차로 확정하면 된다.
